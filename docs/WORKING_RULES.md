# Working Rules

1. Do not change unrelated UI or architecture unless required
2. Prefer small focused commits
3. Explain model changes before making them
4. Preserve existing behavior unless the spec requires a change
5. If a requirement is ambiguous, stop and ask for clarification before guessing
6. After each task, provide a concise summary of:
   1. files changed
   2. why they changed
   3. what needs testing
